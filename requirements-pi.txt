# Container / Raspberry Pi (32-bit ARM, armv7l) dependency set.
#
# numpy and pandas have NO prebuilt wheels for 32-bit ARM on PyPI, so a normal
# `pip install` tries to compile them from source (the Meson build that fails on
# a Pi). Instead the Containerfile installs these from piwheels.org, which hosts
# prebuilt armv7l wheels — but only for the Python that ships with Raspberry Pi
# OS (3.11 on Bookworm). These versions are chosen for solid piwheels coverage.
#
# The dev environment uses requirements.txt (newer numpy/pandas on Python 3.14);
# this file exists only because cross-arch + cross-Python pins can't be identical.
#
# transformers/torch (FinBERT, in requirements.txt) are intentionally absent:
# torch publishes no 32-bit ARM wheels, so the bot falls back to VADER on the Pi
# (data/finbert.py raises FinBertUnavailable). VADER is pure Python, so it
# installs from PyPI here without needing piwheels.
